## 测试支付的注意事项

### "支付方式已停用"——测试服支付报错的另一原因

FB 收紧了支付测试权限：

- 其他应用身份不再默认拥有测试者权限；
- **必须明确把账号添加到 Payments Account 测试者白名单**；
- 以测试者身份触发支付，才能使用通用测试银行卡（如 `4242 4242 4242 4242`）支付。

参考同目录文档：

- 支付白名单配置说明：[FB Canvas 沙盒支付](../其他/FB%20/FB%20Canvas%20沙盒支付.md)
- fbId 获取方法：[如何获取 fbId](../其他/FB%20/如何获取%20fbId.md)

### 排查"支付方式已停用"的步骤

| 怀疑维度 | 验证方式 |
|--------|---------|
| 账号问题 | 换账号试试 |
| 支付卡问题 | 换支付卡试试 |
| FB 服务问题 | 过一段时间再试，持续监测 |

### 测试卡 vs 真实卡

- **测试服**：测试卡 `4242 4242 4242 4242`（白名单账号）；
- **正式服**：真实卡，由财务流程申请，仅允许特定设备在财务室当场绑定后支付。

## 经验沉淀

1. **Canvas 架构属于"维护降级"状态**：旧接口偶发长时间不可用是常态，不要默认 FB 服务永远在线；
2. **测试服报"支付方式已停用"**：先确认测试账号是否在 Payments Account 白名单内，再排查其他维度；
3. **引流策略**：不依赖 FB SDK 升级，而是通过自建 Web 端 + AppCharge 长期化退路；
4. **登录方式**：当前 PID + Email 双因子登录是 CV 时期临时方案，没有 Email 的玩家通过 inbox 引导绑定邮箱后再迁移。
