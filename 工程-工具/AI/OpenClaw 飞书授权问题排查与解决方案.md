# OpenClaw 飞书授权问题排查与解决方案

## 问题背景

在使用 OpenClaw + 飞书机器人时，AI 读取消息分为两种模式，权限要求不同：

---

## ✅ 被动接收（不需要授权）

以下情况飞书事件会把内容**直接带入 OpenClaw 会话上下文**，LLM 可直接处理，无需额外授权：

- 消息中有明确的**消息引用**（reply）
- 消息中有 **@机器人** 的记录
- 与机器人的**直聊（私聊）**

> 原理：飞书事件推送会把相关消息内容直接带到 OpenClaw，成为会话上下文，LLM 直接处理即可。

---

## ❌ 主动拉取（需要用户 OAuth Token）

以下情况需要调用飞书 API 主动拉取数据，必须以**用户身份**完成 OAuth 授权：

- 未指定明确的消息记录，需要**读取历史消息**
- 需要**搜索聊天记录**
- 其他需要以用户身份发起的 API 请求

> 问题根因：`/feishu auth` 命令只有 **App Owner** 能执行，普通用户无法完成授权，导致卡住。

---

## 解决方案：创建独立飞书机器人应用

### 步骤

1. **以自己的身份创建飞书机器人应用**
   - 登录飞书开放平台：https://open.feishu.cn/app
   - 创建新的自建应用

2. **配置 OpenClaw**
   - 将新应用的 `appId` / `appSecret` 配置到 OpenClaw
   - OpenClaw 的飞书插件支持配置多个账号

3. **完成用户配对**
   ```bash
   openclaw pairing approve feishu <your_open_id>
   ```
   以自己的飞书 `open_id` 完成配对

4. **完成 OAuth 授权**
   - 以自己为 App Owner 执行 `/feishu auth` 完成授权

5. **配置事件与回调**
   - 进入新建的飞书机器人应用
   - 左侧菜单 → **事件与回调**
   - 订阅方式改为 **「使用长链接接收事件」**（⚠️ 不是 Webhook）
   - 点击验证 → 确认保存

6. **测试与授权**
   - 测试连接（机器人回复即成功）
   - 完成必要的飞书 API 授权（飞书卡片形式推送，按提示操作）

7. **发布新的应用版本**
   - 提交审核并发布，确保权限配置生效

---

## 关键区别总结

| 场景 | 是否需要授权 | 原理 |
|---|---|---|
| 回复引用的消息 | ❌ 不需要 | 飞书事件直接带入上下文 |
| @机器人发消息 | ❌ 不需要 | 飞书事件直接带入上下文 |
| 与机器人私聊 | ❌ 不需要 | 飞书事件直接带入上下文 |
| 读取群历史消息 | ✅ 需要 | 调用飞书 API，需用户 OAuth Token |
| 搜索聊天记录 | ✅ 需要 | 调用飞书 API，需用户 OAuth Token |
