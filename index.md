## 📝 文档管理规范

### 文档编写规范

1. **文档分类要求**
   - 按功能模块组织文档
   - 使用清晰的目录结构
   - 保持文档命名一致性

2. **内容规范**
   - 使用标准 Markdown 语法
   - 代码示例需可执行
   - 图片资源统一管理
   - 保持更新时效性

3. **命名规范**
   - 中文文档名称直接使用中文
   - 英文文档使用小写+连字符
   - 避免使用特殊字符

### 新增文档流程

1. **确定文档位置**
   - 根据内容选择合适的目录
   - 遵循现有的组织结构

2. **创建文档文件**
   ```bash
   # 在相应目录创建 .md 文件
   touch docs/分类/新文档.md
   ```

3. **编写文档内容**
   - 添加文档标题和描述
   - 组织章节结构
   - 添加必要的代码示例

4. **更新与提交**
   ```bash
   git add .
   git commit -m "docs: 添加新文档"
   git push
   ```

### 文档维护要求

- **定期审查**：每月检查文档准确性
- **及时更新**：功能变更时同步更新文档
- **版本标注**：重要变更需标注版本
- **链接检查**：确保所有链接有效

## ⚙️ 侧边栏配置说明

文档侧边栏通过 `.vitepress/sidebar.mjs` 自动生成，支持以下配置：

### 目录排序
```javascript
const directoryOrder = {
  '活动': 1,    // 数字越小越靠前
  '关卡': 2,
  'Native': 3,
  '协议': 4,
  '工具': 5,
  '其他': 6,
}
```

### 忽略列表
```javascript
const ignoreList = [
  '/其他/测试文档.md',  // 忽略特定文件
  'README.md',          // 忽略所有 README.md
  '/临时目录/',         // 忽略整个目录
  'draft-',            // 忽略 draft- 开头的文件
]
```

### 名称映射
特殊文件名会自动转换为友好显示名称：
- `index.md` → "概览"
- `README.md` → "说明"

## 🚦 使用注意事项

### 本地开发

1. **热更新说明**
   - 文档内容修改：自动热更新
   - 新增/删除文档：需重启服务
   - 配置文件修改：需重启服务

2. **重启服务方法**
   - VS Code：在终端找到运行任务，按 `h` 然后 `r`
   - 手动：`Ctrl+C` 停止，重新运行 `npm run dev`

### 协作开发

1. **分支管理**
   - 文档修改直接在 main 分支
   - 大型重构创建 feature 分支

2. **提交规范**
   - 使用 `docs:` 前缀标识文档提交
   - 描述清晰的修改内容

3. **冲突处理**
   - 优先保留最新内容
   - 确保格式正确性
